# Veromnuka visual system

Этот файл фиксирует визуальную систему сайта Veromnuka. Его нужно учитывать при любых правках дизайна, верстки, анимаций, блоков, карточек и кейсов.

## Главная идея сайта

Veromnuka собирает хаос идей в бренд-систему.

Сайт должен выглядеть как дизайнерская bento-панель: много смыслов, карточек, визуалов и направлений, собранных в понятную систему.

Это сайт бренд-стратега и дизайнера, который работает с продуктами, упаковкой, FMCG, B2B-продажами, выставками, ритейлом, e-commerce и материалами для рынка.

## Общий визуальный вайб

Сайт должен выглядеть как чистая дизайнерская bento-панель: Inter, рубиновый акцент, пыльно-голубые визуальные поля и аккуратная структура без случайного визуального мусора.

Сайт должен выглядеть:
- дизайнерским;
- редакционным;
- живым;
- немного дерзким;
- визуально собранным;
- с ощущением аккуратной 3D-глубины;
- ближе к дизайнерской доске, FMCG-упаковке и технической маркировке, чем к SaaS-лендингу.

Сайт не должен выглядеть как обычная скролльная презентация.

## Bento-логика

Главный визуальный язык сайта: bento-сетка.

Использовать:
- карточки разного размера;
- крупные визуальные окна;
- маленькие стикеры;
- широкие плашки;
- вертикальные карточки;
- плотную, но аккуратную сетку;
- единую систему крупных скруглений;
- светлый пыльный фон;
- тени и аккуратную глубину;
- пыльно-голубые визуальные поля;
- точечные рубиновые акценты;
- упаковочную маркировочную графику.

Карточки могут быть разного масштаба, если это помогает сканировать сайт и показывать диапазон работ.

## Типографика

Основной шрифт сайта:
PP Neue Machina.

Использовать для:
- body;
- навигации;
- кнопок;
- поиска;
- карточек;
- форм;
- подписей;
- тегов;
- абзацев;
- обычных интерфейсных текстов.

Display-шрифт:
Inter Tight Bold.

Использовать только для:
- H1;
- H2 основных секций;
- крупных display-заголовков;
- крупных акцентных заголовков секций.

Old Soviet остается локальным файлом в папке `fonts`, но не применяется к H1, H2, крупным bento-заголовкам и видимым крупным заголовкам. Если подключение `@font-face` сохранено в CSS, переменная display-шрифта должна вести на Inter Tight, а не на Old Soviet.

Настройки для Inter Tight Bold:
- H1: line-height 0.82, letter-spacing -0.09em;
- H2: line-height 0.95, letter-spacing -0.06em;

Настройки для PP Neue Machina:
- letter-spacing -0.08em допустим для навигации, кнопок, поиска, карточек, тегов, подписей и коротких интерфейсных текстов;
- для длинных абзацев можно ослаблять трекинг, если читаемость заметно ухудшается.

На мобильной версии проверять переносы. Заголовки не должны слипаться или вылезать за экран.

## Цвет и акценты

Новая палитра:
- `#F3F3F3` - основной фон сайта, примерно 50-60% визуальной площади;
- `#C3CFE3` - пыльно-голубой заполняющий цвет для крупных визуальных полей, плейсхолдеров, bento-зон и карточек с изображениями, примерно 20-25%;
- `#2B2A2A` - основной темный цвет для текста, рамок, кнопок, иконок и графических линий, примерно 10-15%;
- `#930D0A` - рубиновый акцент для стикеров, маленьких знаков, декоративных элементов, активных состояний, hover и отдельных плашек, примерно 8-12%.

Рубиновым цветом не заливать большие блоки. Крупные поверхности держать светлыми или пыльно-голубыми, а темный использовать как графический каркас.


## Система скруглений

Использовать единые переменные:
- `--radius-card: 36px` для карточек, bento-блоков, кейсов, направлений, визуальных плейсхолдеров и карточки “Вероника Житникова”;
- `--radius-hero: 48px` для крупных визуальных блоков первого экрана;
- `--radius-small: 18px` для маленьких стикеров, тегов, внутренних плашек и декоративных элементов;
- `--radius-pill: 999px` для кнопок, поиска, вытянутых фильтров и pill-элементов.

## Упаковочная маркировочная графика

Будущий декоративный слой должен быть отдельным осмысленным SVG-набором упаковочной графики.

Текущие случайные декоры, фоновая клетка, blob-формы, звездочки, вертикальные подписи и псевдо-маркировки удалены из интерфейса, чтобы вернуть читаемость и чистую базовую структуру.

Когда появится утвержденный SVG-набор, маркировки должны работать только как декоративные отсылки к языку упаковки. Они не должны выглядеть как реальная сертификация конкретного продукта. Основной цвет будущей графики - `#930D0A`, дополнительно можно использовать `#2B2A2A` для тонких линий и белый или полупрозрачный светлый цвет на пыльно-голубых блоках `#C3CFE3`.

## 3D

3D является частью визуального языка Veromnuka.

Использовать 3D осмысленно:
- упаковка;
- мокапы;
- выставочные стенды;
- продуктовые сцены;
- AI-визуалы;
- объемные карточки;
- легкий tilt при hover;
- глубина и тени.

3D подходит сайту, потому что Veromnuka работает с упаковкой, стендами, визуализацией продукта и пространственными материалами.

## Motion

Карточки должны отзываться при наведении.

Подходящие эффекты:
- легкий подъем карточки;
- усиление тени;
- мягкий glow;
- небольшое увеличение картинки внутри карточки;
- аккуратный desktop tilt;
- плавное появление при скролле;
- движение стикеров;
- мягкие переходы.

На мобильных устройствах tilt отключать. Оставлять только легкое взаимодействие.

Учитывать prefers-reduced-motion.

## Кейсы

Кейсы должны быть большой визуальной витриной.

Один клиент может давать несколько карточек:
- дизайн упаковки;
- дизайн-система бренда;
- выставочный стенд;
- презентация;
- каталог;
- B2B-сайт;
- AI-визуалы продукта;
- маркетинговые фото продукта.

Единица галереи: отдельная задача, а не клиент целиком.

Карточка кейса должна быстро считываться:
- бренд;
- направление;
- короткая бизнес-задача;
- сильное превью.

Глубокий переход внутрь карточки вторичен. Большинство клиентов сканируют превью и оценивают стилистику.

## Карточка Вероники

Карточка “Вероника Житникова” должна быть первой карточкой в сетке кейсов.

Она должна быть частью сетки, но немного отличаться.

Текст карточки:

Вероника Житникова

Бренд-стратег и дизайнер с опытом более 15 лет.

Ох*енный продукт сам себя не продаст. Чтобы забрать свою долю рынка, ему нужна стратегия, упаковка и ясная форма.

Проект веду лично, команду подключаю под объем задачи.

Кнопка:
Обсудить проект

Ссылка:
#contacts

В верхней части карточки нужен визуальный блок под будущий микрослайдер с фото Вероники за работой.

## SEO

Визуальный сайт должен сохранять SEO-структуру.

Сохранять:
- один H1;
- понятные H2 у секций;
- текстовые названия кейсов;
- подписи к карточкам;
- alt для изображений;
- ключевые слова в SEO-названиях и описаниях кейсов;
- нормальную HTML-структуру.

В кейсах использовать поисковые формулировки:
- дизайн упаковки;
- упаковка FMCG;
- дизайн упаковки продуктов питания;
- рыбная продукция;
- дизайн-система бренда;
- презентация для продаж;
- B2B-сайт производителя;
- выставочный стенд;
- AI-визуализация продукта;
- маркетинговые фото продукта.

## Запреты

Не добавлять:
- SMM как отдельную услугу;
- маркетплейсы как отдельную услугу;
- интернет-магазины;
- продакшен как отдельную услугу;
- случайные стоковые фото;
- внешние шрифты через CDN;
- Google Fonts;
- тяжелые библиотеки без причины.

Не менять утвержденные тексты без отдельной команды.

Не использовать в видимых текстах:
- длинное тире U+2014;
- букву е с точками, кроме имен собственных и цитат;
- конструкцию “не ..., а ...”;
- пустые фразы вроде “комплексный подход”, “точка роста”, “уникальная возможность”, “важно отметить”.

## Как работать с Codex

Каждая новая задача по дизайну должна учитывать этот файл.

Перед правками:
- проверить текущую структуру сайта;
- не откатывать уже утвержденные блоки;
- не ломать поиск;
- не ломать карточку Вероники;
- не удалять SEO-тексты;
- не создавать конфликтные PR поверх старых веток.

После правок:
- проверить HTML;
- выполнить node --check script.js;
- выполнить node --check opportunities.js;
- выполнить git diff --check;
- проверить отсутствие U+2014 в index.html, styles.css, script.js, opportunities.js;
- написать в Summary, какие визуальные правила из DESIGN.md были использованы.
